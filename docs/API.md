# API Reference

The full interactive API documentation is served by the running server:

- **Swagger UI**: <http://localhost:4000/api-docs>
- **OpenAPI JSON**: <http://localhost:4000/api-docs-json>

The spec is generated by `swagger.js` using `swagger-jsdoc`.

## Quick reference

| Group | Endpoints |
|---|---|
| Health | `GET /ping`, `GET /health/db` |
| Auth | `POST /usuarios/register`, `POST /usuarios/login`, `POST /usuarios/auth/google`, `DELETE /usuarios/auth/google/unlink`, `POST /usuarios/restablecer-solicitud`, `POST /usuarios/restablecer` |
| Users | `GET /usuarios/perfil`, `PUT /usuarios/perfil`, `DELETE /usuarios/perfil`, `GET /usuarios/all` (admin), `DELETE /usuarios/admin/:id` (admin) |
| Productos | `GET /productos`, `GET /productos/:id/stock`, `POST /productos` (admin), `PUT /productos/:id` (admin) |
| Reservas | `POST /productos/:id/reservar`, `PUT /productos/reserva/:reservaId/confirmar`, `DELETE /productos/reserva/:reservaId/cancelar`, `GET /productos/mis-reservas` |
| Citas | `GET /citas`, `POST /citas`, `PUT /citas/:id`, `DELETE /citas/:id`, `GET /citas/admin` |
| Eventos | `GET /eventos`, `POST /eventos` (admin), `PUT /eventos/:id` (admin), `DELETE /eventos/:id` (admin), `POST /eventos/inscripcion`, `DELETE /eventos/inscripcion/:evento_id` |
| Blog | `GET /blog`, `GET /blog/:id`, `POST /blog` (admin), `PUT /blog/:id` (admin), `DELETE /blog/:id` (admin) |
| FAQ | `GET /faq`, `POST /faq` (admin), `PUT /faq/:id` (admin), `DELETE /faq/:id` (admin) |
| Testimonios | `GET /testimonios`, `POST /testimonios`, `PUT /testimonios/:id`, `DELETE /testimonios/:id`, `PUT /testimonios/:id/aceptar` (admin), `PUT /testimonios/:id/rechazar` (admin), `GET /testimonios/pendientes` (admin) |
| Personalización | `POST /personalizacion`, `GET /personalizacion` (admin), `PUT /personalizacion/:id/aceptar` (admin), `PUT /personalizacion/:id/rechazar` (admin) |
| Disponibilidad | `GET /disponibilidad/publicas`, `GET /disponibilidad/publicas-con-cita`, `GET /disponibilidad/admin` (admin), `POST /disponibilidad` (admin), etc. |
| Servicios | `GET /servicios` |
| Reportes | `GET /reporte/auditoria` (admin), `GET /reporte/usuarios/activos`, etc. |
| Chat | `POST /chat` |

## Status codes

See `docs/API_STATUS_CODES.md` for the full per-endpoint status code matrix.

## Authentication

Most endpoints require `Authorization: Bearer <jwt>` or `x-auth-token: <jwt>` header.
